About this Backend

The Bluemix Backend for Frontend template demonstrates how to create and deploy omnichannel application to the IBM Bluemix Cloud Platform. The project has the the ability to browse a list of products and supports the viewing of images for each item in the product list.

A "Backend for Frontend" application defines different backend services and capaiblities for different frontend clients. Using this architecture, developers can expose custom APIs from their Compute to deliver client solutions with a scalable cloud backend.

This lightweight Node backend uses Bluemix Mobile services, Cloudant NoSQL DB, Object Storage, and a LoopBack runtime to expose a Swagger definition. Using the Bluemix, we can connect this Compute to our project, generate a SDK, plug the APIs into mobile or web clients, and much more.

The following sections explain what each of these services do, how they are implemented in this backend, and where to get more detailed information about the services from the official documentation.


  • LoopBack

    The LoopBack framework is a set of Node.js modules that you can use independently or together to quickly build REST APIs. A LoopBack application interacts with datasources through the LoopBack model API, available locally within Node.js, remotely over REST, and via native client APIs for iOS, Android, and HTML5. Using these APIs, apps can query databases, store data, upload files, send emails, create push notifications, register users, and perform other actions provided by datasources and services.

    In this backend, we use LoopBack for a single point of REST integration for our Cloudant NoSQL DB datasource and images stored on Object Storage. Click the View API Reference button to browse the API for the Bluemix Backend for Frontend template.

    For more information about LoopBack, see the official documentation below.

  • Cloudant NoSQL DB

    Cloudant NoSQL DB provides access to a fully managed NoSQL JSON data layer that's always on. This service is compatible with CouchDB, and accessible through a simple to use HTTP interface for mobile and web application models.

    In this backend, we provision a Cloudant NoSQL DB service instance and supply that with data from a products.json file which contains a list of products for the Backend for Frontend template.

    If you open the Cloudant NoSQL DB service instance, you can browse the data inside the template. Each document contains information about the price, rating, description, etc. for each article of clothing that is available from the store.

    For more information about Cloudant NoSQL DB, see the official documentation below.

  • Object Storage

    Object Storage provides an unstructured cloud data store to build and deliver cloud applications and services with lowered cost, reliability, and speed to market. Bluemix developers and users can access and store unstructured data content and can interactively compose and connect to applications and services. The Object Storage service also provides programmatic access via API, SDKs and a consumable UI for object management.

    In this backend, we provision a Object Storage service instance and supply that with data from a clothes folder which contains the images for the template.

    If you open the Object Storage service instance, you can browse the data inside the template. The clothes container contains the images of each item available.

    For more information about Object Storage, see the official documentation below.

  • Mobile Client Access

    Finely tune mobile apps with operational analytics from real time performance and data usage. Ensure communications with your backend systems are secure. Let your app users to log in with custom accounts or existing social accounts.

    In this backend, we protect an endpoint that is only accessible after authentication with the Mobile Client Access service from the mobile application.

    For more information about Mobile Client Access, see the official documentation below.

  • Push Notifications

    The Push Notification service provides a unified push service to send real-time notifications to mobile and web applications. The service provides the ability to personalize and send notifications to a segment of users, single user or broadcast to all users.

    In this backend, we provision a Push Notifications service instance for sending push notifications to a mobile or web app.

    For more information about Push Notifications, see the official documentation below.

  • Mobile Analytics

    Mobile Analytics for Bluemix provides developers, IT administrators, and business stakeholders insight into how their mobile app is performing and how it is being used. Monitor performance and usage of all your applications from your desktop or tablet. Quickly identify trends and anomalies, drill down to resolve issues and trigger alerts when key metrics cross critical thresholds.

    In this backend, we use Mobile Analytics to view device sessions, monitor crashes, and create custom charts.

    For more information about Mobile Analytics, see the official documentation below.
